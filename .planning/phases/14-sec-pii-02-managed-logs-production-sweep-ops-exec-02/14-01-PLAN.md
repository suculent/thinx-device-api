# Plan 14-01 — SEC-PII-02 managed_logs Production Sweep (OPS-EXEC-02)

## Objective

Close OPS-EXEC-02 by emitting a GSD CHECKPOINT for the operator to run the documented `managed_logs` redaction sweep against production CouchDB per `.planning/runbooks/managed-logs-redaction.md`. On resume, verify operator-supplied sentinel inputs, append Execution Annex to the runbook, flip OPS-EXEC-02 to Verified, emit SUMMARY.

## Tasks

### Task 1: CHECKPOINT — operator runs production sweep

**Type:** checkpoint:human-verify (blocking)
**Read first:** `.planning/runbooks/managed-logs-redaction.md` (canonical procedure).

**Action:** Emit `## CHECKPOINT REACHED` with the full operator instruction set (D-13). Wait for operator to return sentinel inputs.

**Sentinel inputs (D-03) the resume executor will verify:**
1. `.planning/phases/14-.../dry-run.log` — non-empty; contains `[scan] scanned=` and `[scan] dirty=` lines; dirty count in expected range (not ≈0, not >700k).
2. `.planning/phases/14-.../apply.log` — non-empty; contains `[apply] done touched=` and `[apply] done conflicts=` lines; snapshot path included.
3. `.planning/phases/14-.../sample.log` — non-empty; contains `[sample] PASS:` line; exit code 0 recorded.
4. Operator resume signal carries: timestamp UTC, operator initials, snapshot path + SHA256, pre/post `disk_size` figures from CouchDB `_info`, Slack receipt confirmation (channel + message timestamp or "manual paste" if OBS-01 didn't fire).

**Verify:** All 4 sentinel inputs present and content-valid. If missing → re-emit CHECKPOINT.
