<objective>
Build `scripts/redact-managed-logs.js` — the operator-side CLI that streams CouchDB `managed_logs` in pages of 1000 docs, scans each doc's `message` (and any string field) for the SEC-PII-01 leak shape (raw 64-char hex `reset_key` substrings + raw email patterns + activation/mailgun tokens), and overlays `[REDACTED-RESET_KEY]` / `[REDACTED-EMAIL]` via `_bulk_docs` batched in groups of 500. Default mode is dry-run; `--apply` is destructive and gated behind a mandatory `--snapshot-to <path>` JSONL forensic dump. A `--sample N` subcommand provides the SEC-PII-02 verification primitive (zero raw-key/email patterns in N random recent + N random old docs post-remediation). A `--max-docs N` flag caps the run for staged rollouts. Ship with a fixture-based Jasmine spec (`spec/jasmine/ZZ-RedactionScriptSpec.js`) — NO live CouchDB dependency, so CI green-gates pass in the standard sandbox.

Purpose: SEC-PII-02 mandates a sampling-verified zero-leak post-state on ~658k pre-Phase-2 audit docs. The script is the operator's working surface; the unit spec proves correctness without needing production access. The redaction is destructive of audit-log CONTENT (the original PII text is overwritten); the snapshot is the only forensic-rollback artifact, so the snapshot-required gate is non-negotiable. The script reuses `Util.redactToken` / `Util.redactEmail` from `lib/thinx/util.js` so the regex stays single-source-truth with SEC-PII-01 call sites in `owner.js`.

Output: One new script (`scripts/redact-managed-logs.js`, ~200-300 lines including help/usage), one new spec (`spec/jasmine/ZZ-RedactionScriptSpec.js`). One atomic GPG-signed commit: `feat(SEC-PII-02): managed_logs PII redaction script with sampling verification`.

Create the operator CLI as a single Node.js script (no bundler). Honor these invariants drawn from CONTEXT.md per D-01 (hybrid strategy):

1. CLI surface (minimal arg parser — no external dep; use process.argv directly to stay zero-new-deps):
   - `--scan` (default if no mode flag): dry-run. Streams docs in pages of 1000 (via `nano.list({include_docs: true, limit: 1000, startkey_docid: ...})`), counts how many docs match raw 64-char hex reset_key regex AND/OR raw email regex in `message` or `flags[*]`. Emits a summary to stdout: total scanned, total matching, breakdown by leak-shape, sample first-N redacted message previews.
   - `--apply`: destructive. REFUSES to run unless `--snapshot-to <path>` is also passed AND the path is non-empty (script writes JSONL of every doc it touches BEFORE issuing any `_bulk_docs` POST). On refuse, print error referencing CONTEXT.md invariant and exit with non-zero code (e.g., 64).
   - `--snapshot-to <path>`: mandatory companion to `--apply`. Script opens an append-mode write stream and pipes each touched doc as JSONL BEFORE the corresponding `_bulk_docs` batch fires. If the path's parent dir doesn't exist, refuse to start (exit non-zero).
   - `--sample <N>`: verification subcommand. Picks N random docs from the "recent" half (sorted by `_id` descending) + N random docs from the "old" half (ascending), scans for raw 64-char hex reset_key + raw email patterns in `message` and any string field, exits 0 if zero leaks found across all 2N docs, exits non-zero (e.g., 65) if any leak detected. Print which docs leaked (by `_id` only, NEVER the leak content itself — the operator can re-fetch if needed).
   - `--max-docs <N>`: cap total docs touched per invocation. Default unbounded. Useful for staged rollouts.
   - `--batch-size <N>`: docs per `_bulk_docs` request. Default 500.
   - `--db-name <name>`: override the default `${THINX_PREFIX}managed_logs`. Default reads from `process.env.THINX_PREFIX` + `managed_logs` suffix.
   - `--help` / `-h`: print usage and exit 0.

2. Credential resolution (per CONTEXT.md constraint — ZERO creds in repo):
   - Read `process.env.COUCHDB_URL`, `process.env.COUCHDB_USER`, `process.env.COUCHDB_PASSWORD`. Compose nano connection: if URL contains creds, use as-is; otherwise inject `${USER}:${PASSWORD}@` between scheme and host.
   - If any required env var is unset, refuse to run with a clear error (exit non-zero). Exception: `--help` runs without creds.

3. Redaction core (the per-doc transform):
   - Define module-level constants:
     - `RESET_KEY_REGEX = /[0-9a-f]{64}/g`
     - `EMAIL_REGEX = /[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}/g`
   - Export `containsRawPII(doc)` (boolean): scans `doc.message` (string), `doc.flags` (array of strings), and any other string-valued field for either regex match.
   - Export `redactDoc(doc)` (returns `{ changed: boolean, doc: <new doc with redacted strings> }`): clones the doc, replaces RESET_KEY_REGEX matches with literal `[REDACTED-RESET_KEY]` and EMAIL_REGEX matches with literal `[REDACTED-EMAIL]` in all string-typed fields (recursive over `flags` array). Preserves `_id` and `_rev` (CouchDB requires `_rev` in `_bulk_docs` updates). Sets a new field `redacted_by: "SEC-PII-02"` and `redacted_at: <ISO timestamp>` for forensic audit.
   - Export `scanDoc(doc)` (returns `{ resetKeyHits: number, emailHits: number, fields: string[] }`) — used by `--sample` and `--scan`.

4. Pagination + batching (the per-DB loop):
   - Stream pages of 1000 docs via `nano.list({include_docs: true, limit: 1000, startkey_docid: <last_seen_id_or_null>})`. Track the last `_id` to advance startkey_docid; halt when a page returns fewer than `limit` rows.
   - Accumulate dirty docs (those where `redactDoc(d).changed === true`) into a batch buffer. When buffer length >= `--batch-size`, write each dirty doc as one JSONL line to snapshot stream FIRST (await flush), THEN POST batch to `_bulk_docs`. Drain on EOF.
   - Respect `--max-docs`: halt the loop once N docs have been touched (apply mode) or scanned (scan mode). The cap counts SCANNED docs in scan mode, TOUCHED docs in apply mode.

5. Idempotency guarantee:
   - The redactor replaces raw matches with `[REDACTED-RESET_KEY]` / `[REDACTED-EMAIL]` (LITERAL strings, NOT regex-matchable as 64-char hex or email). A second `--apply` run will find zero matches and produce zero `_bulk_docs` writes. The redacted-marker strings deliberately contain `[`, `]`, and uppercase letters so they CANNOT match `/[0-9a-f]{64}/` or the email regex.

6. Module exports:
   - Top-level if `require.main === module` block to run the CLI. Otherwise export `{ redactDoc, scanDoc, containsRawPII, RESET_KEY_REGEX, EMAIL_REGEX }` so the spec can require it directly without invoking the CLI.

7. Logging hygiene per SEC-PII-01:
   - Never log raw PII to stdout/stderr. Use `Util.redactToken(matchedString)` and `Util.redactEmail(matchedString)` for any progress prints (require `../lib/thinx/util.js`). Doc `_id`s ARE safe to log (they are mtimes per audit.js, not PII).
   - On `--apply` mode, prefix every log line with timestamp + mode flag so operator logs are forensically reconstructable.

8. Help text: print a usage block with all flags, a worked example for `--scan`, `--apply --snapshot-to`, and `--sample`, plus a prominent WARNING about the destructiveness of `--apply` and the snapshot requirement.
