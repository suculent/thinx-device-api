slack-notify v2.0.7 send-contract (verified against `node_modules/slack-notify/src/cjs/index.js`):
- `pub.send(options)` returns `pub.request(data)` which returns a `new Promise((resolve, reject) => { ... })`
- Therefore `.send()` returns a Promise that may resolve OR reject. The caller MUST handle BOTH sync throws (rare, e.g., synchronous TypeError on a malformed `options`) AND async rejections (the common path: `reject(NO_URL_ERROR)`, `reject(resBody)` on non-`"ok"` response, network errors).
- Implication: A bare `try/catch` around `SlackNotify(webhook).send(msg)` catches ONLY the sync path. A `.catch(...)` chained on the returned Promise is required for the async path.
