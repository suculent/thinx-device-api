<objective>
Replace the single non-strict comparison `reset_key != user_reset_key` at `lib/thinx/owner.js:492` with the strict-equality form `reset_key !== user_reset_key` (REFACTOR-02). Scope is strictly the `password_reset` method body (lines :459-500). Add a unit test covering the string-vs-number coercion case the `!=` was implicitly accepting.

Purpose: Close the type-coercion bypass surface in the password-reset key compare. Today's API entry points (`req.body.reset_key`, `req.query.reset_key`) already pass strings, so this is a hardening fix — no functional change for currently-valid inputs. Critically: `!=` would have considered the string `"123"` equal to the number `123`, which is exactly the kind of coercion bug strict equality eliminates. The fix is security-positive without being behavior-breaking for the legacy console flow.

Output: One source edit in `lib/thinx/owner.js` (single character: `!=` → `!==`), one new unit spec block in `spec/jasmine/02-OwnerSpec.js`, one atomic commit: `refactor(REFACTOR-02): strict equality in Owner.password_reset`.
</objective>
