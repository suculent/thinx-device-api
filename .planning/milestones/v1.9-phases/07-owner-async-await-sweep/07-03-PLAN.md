Async/await target shape:
  async delete(owner, callback, res) {
    try {
      await this.userlib.atomic("users", "edit", owner, { deleted: true });
      alog.log(owner, "Owner state changed to deleted.", "warning");
      return callback(res, true, "deleted");
    } catch (a_error) {
      // FALLBACK: try hard destroy
      try {
        await this.userlib.destroy(owner);
        alog.log(owner, "Profile update failed.", "error");
        alog.log(owner, "Owner document destroyed.", "warning");
        return callback(res, true, "deleted");
      } catch (error) {
        alog.log(owner, "Profile update failed.", "error");
        return callback(res, false, "delete_failed");
      }
    }
  }

Note: original code emits `alog.log(owner, "Profile update failed.", "error")` BEFORE testing the destroy result. In the async form, that log line MUST still fire when the atomic fails (before attempting destroy) — preserve by placing it inside the catch (a_error) block BEFORE the inner try. Or place it INSIDE the destroy try/catch as the FIRST line (matches original ordering). Choose the form that preserves the exact alog.log sequence the original emits.
