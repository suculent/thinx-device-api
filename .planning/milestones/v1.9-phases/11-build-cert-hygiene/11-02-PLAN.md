```
/**
 * probeCaFreshness(certPath, caPath)
 *
 * DETECT-only probe — does NOT mutate any cert file.
 *
 * @param {string} certPath  Absolute path to the leaf cert PEM (typically app_config.ssl_cert).
 * @param {string} caPath    Absolute path to the ca.pem PEM bundle (typically app_config.ssl_ca).
 * @returns {object}         { ok, leafIssuer, caContains, message }
 *
 *   ok          {boolean}    true when ca.pem contains the leaf's issuer CN; false on mismatch.
 *   leafIssuer  {string|null} Leaf cert's Issuer.CN (e.g., 'R13'), or null if it could not be parsed.
 *   caContains  {string[]}    Array of Subject.CN values found in the ca.pem chain (e.g., ['R10']).
 *   message     {string|null} null when ok=true; on ok=false, a human-readable message naming the
 *                             expected issuer and what was actually found, with a remediation hint
 *                             pointing at https://letsencrypt.org/certs/.
 *
 * Edge cases (all return ok:false with a descriptive message — NEVER throw):
 *   - leaf or ca file does not exist on disk
 *   - leaf or ca file is malformed PEM (node-forge throws on parse)
 *   - leaf Issuer.CN is not one of R10/R11/R12/R13/R14 (non-LE leaf — return ok:true with a
 *     message='non-LE leaf — probe skipped' note; this matches existing thinx-core.js tolerance
 *     behavior where non-LE certs are not subject to the rotation-tolerance branch)
 *
 * Behavioral guarantee: probeCaFreshness MUST NOT write to disk and MUST NOT call console.log,
 * console.warn, rollbar, or any logging primitive. The caller decides how to surface the result.
 */
function probeCaFreshness(certPath, caPath) { ... }

module.exports = { probeCaFreshness };
```

Existing matcher logic to MIRROR (thinx-core.js:67-71):

```js
const isSupportedLetsEncryptIssuer = (attributes = []) => {
  const organization = getCertAttribute('organizationName', attributes) || getCertAttribute('O', attributes);
  const commonName = getCertAttribute('commonName', attributes) || getCertAttribute('CN', attributes);
  return organization === "Let's Encrypt" && ['R10', 'R11', 'R12', 'R13', 'R14'].includes(commonName);
};
```

Duplicating (not importing) is intentional per D-02: keep the probe self-contained so it has no dependency on thinx-core.js's bootstrap state. The duplicated allowlist is a single line and the cost of duplication is low; the alternative (extracting to a shared module) would re-shape thinx-core.js, violating the "additive only" constraint.
